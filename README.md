## Project Overview

The **Social Spark App** is a mobile application developed using **Kotlin** in **Android Studio**. The app helps users maintain social connections by suggesting small social actions based on the time of day.

This application was created to meet assignment requirements, including building a functional Android app, using conditional logic (if and else if statements), and implementing GitHub version control with GitHub Actions for automated builds.

---

## Purpose and Features

### Purpose:
The main goal of this app is to help users stay socially connected throughout the day by providing quick and simple interaction ideas based on the time entered.

### Key Features:
- **Time-Based Input**: Users can enter a time of day (e.g., Morning, Afternoon, Dinner)
- **Smart Suggestions**: The app provides a relevant social interaction idea
- **Reset Functionality**: Clears input and results easily
- **Error Handling**: Displays helpful messages for invalid or empty input
- **Logging**: Uses Logcat to track user input and app behavior

These features aim to provide an easy and engaging way for users to maintain social relationships.

---

## Design Considerations

The design of the **Social Spark App** was based on the following:

1. **User Experience (UX)**: A simple and clean interface ensures users can easily interact with the app.
   
2. **Simplicity**: The app focuses only on core functionality using basic Kotlin concepts like if and else if statements.
   
3. **Visual Design**: A modern color scheme was used to improve readability and engagement.
   
4. **Performance**: Lightweight code ensures smooth performance without unnecessary complexity.

---

## GitHub and GitHub Actions

This project was managed using **GitHub** for version control. Code was regularly committed and pushed to track progress and maintain backups.

### GitHub Actions:
GitHub Actions was implemented to automate the build process. This includes:

- Automatically building the app on every push
- Ensuring the app compiles successfully
- Detecting errors early in development

This ensures that the project remains functional and ready for submission at all times.
